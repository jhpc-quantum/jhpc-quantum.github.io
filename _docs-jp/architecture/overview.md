# 概要・目標・要件

JHPC-quantumプラットフォームは、インターネットを含むネットワークで接続された複数量子コンピュータと複数HPCシステム（スパコン）から構成されるシステムで、量子コンピュータとHPCシステムを高度に連携させる量子HPC連携アプリケーションの実行を可能とする。

量子コンピュータは、古典コンピュータ、すなわち従来のデジタル計算機の適切な制御の下に動作するもので、計算機システムとしては量子・古典ハイブリッドシステムとなる。現在の主なソフトウェアの研究は、数十量子ビットの量子コンピュータ実機、もしくは、量子計算シミュレータを用いた研究開発がほとんどであり、量子コンピュータのためのソフトウェア開発および量子コンピュータを制御する古典側のコンピュータの規模は単体のサーバーやPCで行われていることが多い。

これから開拓される量子コンピュータの利用を考えると、量子コンピュータに期待されるアプリケーション領域の多くは、これまでスパコンでの計算能力を用いてサポートされてきたアプリケーション領域であり、スパコン側からは量子コンピュータはこの領域のアプリケーションを大幅に加速する装置として捉えることが適切である。これに類似する技術として、現在多くのスパコンで利用されているのがGPUなどの演算加速装置であるが、スパコンのそれぞれのノードごとに付加されるGPUとは異なり、現時点でのほとんどの量子コンピュータは極低温環境などその設置環境の厳しさから、異なる装置として適切なインターコネクトあるいはネットワークを通して接続される。また、しばらくの間はそのコストから複数のスパコン、あるいはサイトから共有利用されることを想定しなくてはならない。このような観点から、スパコンを代表とするHPCシステムと量子コンピュータの共有利用を可能とする量子・HPC連携システムソフトウェアを開発する。量子コンピュータを整備し、本システムソフトウェアにより、複数のスパコンと複数の量子コンピュータを連携利用する、量子・スパコン連携プラットフォームを構築する。さらに、このプラットフォームをクラウド上に展開し、これから劇的に進歩するポスト５Ｇネットワーク上の高機能なサービスを提供する基盤技術を開発する。

以下にJHPC-quantumプラットフォームの要件についてあげる。

1. HPCプログラムの一部分として、量子プログラムを利用できるようにする。実行においては、量子コンピュータだけでなく、スパコンなどHPCシステムの効率的な利用を可能とする。

1. HPCプログラムだけでなく、これまでのqiskitやTKETなどのpythonのフレームワークで書かれたプログラムなど、通常のサーバーのシングルノードで動作するプログラムの実行を可能とし、これまでの量子コンピュータの利用環境と互換性のあるものとする。

1. 複数の量子コンピュータを複数のHPCシステムおよびサーバー環境から利用できるようにし、我々のユーザー管理のポリシーで運用できるようにする。ユーザーの認証、アカウンティング等についてもサポートする。
